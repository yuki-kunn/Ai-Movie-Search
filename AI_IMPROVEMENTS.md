# AI機能の改善とフールプルーフ

このドキュメントでは、Gemini APIを使用した映画検索機能の改善点を説明します。

## 1. プロンプトエンジニアリングの改善

### 改善前の問題点
- 言語の不一致（日本語の指示 + 英語のフィールド名）
- 具体例の不足
- 出力制約が不明確
- エッジケースへの対応不足

### 改善後の特徴

#### 1.1 英語への統一
AIモデルは英語で訓練されているため、プロンプトを英語に統一することで精度が向上します。

#### 1.2 Few-shot Learning（具体例の提供）
```
Examples:

Input: "感動的な映画"
Output: {"genres": ["drama"], "keywords": ["emotional", "touching", "heartwarming"], "searchQuery": "emotional drama"}

Input: "アクション満載のスリリングな映画"
Output: {"genres": ["action", "thriller"], "keywords": ["action-packed", "thrilling", "intense"], "searchQuery": "action thriller"}
```

3つの具体例を提供することで、AIが期待される出力形式を理解しやすくなります。

#### 1.3 明確な制約と指示
- ジャンルは1-3個まで
- キーワードは2-5個まで
- 検索クエリは50文字以内
- 日本語入力の場合は英語に翻訳
- 曖昧な入力の場合は人気ジャンルを選択

#### 1.4 ロールプレイング
```
You are a movie recommendation expert.
```

AIに明確な役割を与えることで、より一貫性のある応答が得られます。

## 3. 出力の検証とサニタイズ

### 実装内容

#### 3.1 型チェック
- `genres`: 配列であることを確認
- `keywords`: 配列であることを確認
- `searchQuery`: 文字列であることを確認

#### 3.2 長さ制限
- `genres`: 最大3個
- `keywords`: 最大5個
- `searchQuery`: 最大100文字

#### 3.3 正規化
- `genres`: 小文字に統一、空文字を除外
- `keywords`: トリミング、空文字を除外
- `searchQuery`: トリミング

#### 3.4 フォールバック
- `searchQuery`がない場合: `keywords[0]`または元の入力を使用
- `genres`も`searchQuery`もない場合: エラーとして扱う

## 4. エラーハンドリングの強化

### 4.1 タイムアウト処理
```typescript
const timeoutPromise = new Promise((_, reject) => {
  setTimeout(() => reject(new Error('Gemini API timeout')), 30000);
});

const result = await Promise.race([
  model.generateContent(prompt),
  timeoutPromise
]);
```

30秒後に自動的にタイムアウトし、TMDBのみで検索を続行します。

### 4.2 JSON解析エラー
- マークダウンコードブロックの削除
- 解析失敗時の詳細ログ
- 元のレスポンステキストを出力

### 4.3 エラーの分類
1. **タイムアウトエラー**: 30秒以内に応答なし
2. **429エラー**: クォータ超過 → サーキットブレーカー発動
3. **503エラー**: サービス過負荷 → サーキットブレーカー発動
4. **その他**: JSON解析エラー、検証エラーなど

## 5. レート制限

### 実装内容
- **制限**: 1分間に20リクエストまで（IPアドレス単位）
- **ウィンドウ**: 1分
- **応答**: 429 Too Many Requests

### 目的
- DoS攻撃の防止
- サーバーリソースの保護
- APIクォータの節約

## 7. 多層防御（Defense in Depth）

```
ユーザー入力
    ↓
1. 入力サニタイズ（XSS対策、長さ制限）
    ↓
2. レート制限（DoS対策）
    ↓
3. キャッシュチェック（APIコール削減）
    ↓
4. サーキットブレーカー（過負荷時の保護）
    ↓
5. Gemini API呼び出し（タイムアウト付き）
    ↓
6. 出力の検証とサニタイズ
    ↓
7. フォールバック（TMDB直接検索）
    ↓
結果
```

## 8. テスト推奨項目

### 正常系
- [ ] 日本語入力（例: "感動的な映画"）
- [ ] 英語入力（例: "action movie"）
- [ ] 短い入力（例: "comedy"）
- [ ] 長い入力（100文字以上）

### 異常系
- [ ] 空の入力
- [ ] 特殊文字（`<script>alert('test')</script>`）
- [ ] 非常に長い入力（1000文字以上）
- [ ] 連続リクエスト（レート制限のテスト）
- [ ] Gemini APIキーなし

### エッジケース
- [ ] 複数のスペース（例: "action    movie"）
- [ ] 前後のスペース（例: "  comedy  "）
- [ ] 絵文字（例: "😊 happy movie"）
- [ ] 数字のみ（例: "2024"）

## 9. モニタリング推奨項目

本番環境では以下のメトリクスを監視することを推奨します：

- **キャッシュヒット率**: 50%以上が理想
- **Gemini APIタイムアウト率**: 5%未満が理想
- **サーキットブレーカー発動頻度**: 1日1回未満が理想
- **レート制限違反**: 1日10回未満が理想
- **平均レスポンス時間**: 3秒未満が理想
